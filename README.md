O Problema: Identificação de um pico de esforço operacional nas quintas-feiras, com índice de 3,43, representando 50,59% da carga semanal.

A Solução: Implementação de nova lógica de distribuição de tarefas simulada em Python (200k linhas) e processada em SQL.

O Resultado: Redução drástica do esforço para 0,56 na sexta-feira (cenário pós-intervenção), uma melhoria de quase 84% na eficiência.
